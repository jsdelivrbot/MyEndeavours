CodeType

CCTS TypeCore Data Type
CCTS Dictionary Entry Name Code. Type
Description A "Code. Type" is a character string (letters, figures or symbols) that for brevity and/or language independence may be used to represent a definitive value or text. The value or text is part of a finite list of allowed values.
Usage Rule"Code. Type" should be used in case of a finite list of allowed values and the "Identifier. Type" should be used in case of a infinite list of objects.

The "Code. Type" is used for all elements that are used in the communication between partners or systems to enable a common coded value representation. This code list should be relatively stable, and not subject to frequent or significant changes. Typical example of code types are: Country_ Code. Type and  Language_ Code. Type. The "Code. Type" is intended to be used for building BBIEs and BCCs, but not for the definition of codelists.

DateTimeType

CCTS TypeCore Data Type
CCTS Dictionary Entry Name Date Time. Type
Description A "Date Time. Type" is a particular point in time on a particular date.
Usage Rule

The coordinated universal time (UTC) is the standardized basis for time specifications that are used internationally.

Date Time. Type is used for exact time stamps that must contain the day and time. For example, creation date/time, receipt date/time, processing date/time, delivery date/time, and expiry date/time.

Only the Gregorian calendar will be used.

The time part may be expressed with or without the UTC offset.

Attributes

DateType

CCTS TypeCore Data Type
CCTS Dictionary Entry Name Date. Type
Description A "Date. Type" is used to represent a period beginning at midnight and ending next midnight in a given timezone.
Usage Rule

Date. Type should not be used to specify periodic events.

Only the Gregorian calendar should be used.

Date. Type is used when it is only important to know the day in which something occurs, and not the time at which it occurs. When both the date and time are important use Date Time. Type.

IndicatorType

CCTS TypeCore Data Type
CCTS Dictionary Entry Name Indicator. Type
Description An "Indicator Type" is a list of two mutually exclusive boolean values that express the only possible states of a property.
Usage Rule

The allowed values are: "true" and "false". Other values should be locally mapped to the values "true" and "false" in an appropriate manner.

The semantics of the indicator based BBIEs should always consider the boolean concept of "true|false". For example:

Door_ Open_ Indicator. Type (true|false) and not Door_ Position_ Indicator. Type (wich might be "open|close").

Indicator. Type is used for binary classifications, indicators, flags, etc.

MeasureType

CCTS TypeCore Data Type
CCTS Dictionary Entry Name Measure. Type
Description A "Measure. Type" is a numeric value determined by measuring an object along with the specified unit of measure.
Usage Rule

"Measure. Type" must not be confused with "Quantity. Type". The "Measure. Type" should always express a kind of physical dimension such as temperature, length, speed, width, weight, volume, latitude, etc. This semantics should be clearly expressed by the property (term + qualifier) of the specific BBIE.

Typical BBIEs for representation of measures are:

  • Package. Length. Measure
  • Weather. Minimum_ Temperature. Measure

 Typical BBIEs for representation of quantities are:

  • Product. Total. Quantity
  • Item. Minimum_ Shipped. Quantity
  • Shelf. Maximum_ Storage. Quantity

PercentType

CCTS TypeCore Data Type
CCTS Dictionary Entry Name Percent. Type
Description A "Percent. Type" is a number that relates to the comparison figure 100.
Usage RuleNo measurements or currencies are specified in "Percent. Type".

Percent. Type can be used to represent a percentage that indicates how many hundredths of a basic value are to be calculated. The result of the calculation is the proportion in percent of, e.g., amounts, values, rates, discounts, and taxes.

Further examples for the application of Percent are proportion and comparison information, such as dividends and earnings, or a percentage comparison of target and actual business results, or trade or amount margins.

RateType

CCTS TypeCore Data Type
CCTS Dictionary Entry Name Rate. Type
Description A "Rate. Type" is a fraction whose numerator and denominator are quantities, values, or dimensionless factors, independent from each other.
Usage Rule

Rate specifies a rate between two factors with specific units of measure, for example, the daily turnover of a business.

For a purely numerical ratio where the numerator and the denominator are used without units of measure, the Ratio should be used in accordance with UN/CEFACT CCTS
